# 校园外卖信息聚合平台 — 错误码设计

**文档类型**：错误码规范  
**适用范围**：外卖聚合模块（小程序端 + 后台管理端 + 后端服务）

---

## 1. 设计原则

- **统一格式**：所有接口返回体中的 `code` 字段遵循本规范。
- **前后端对齐**：错误码及含义以本文件为唯一依据，前后端均不得私自新增“魔法数字”。
- **分段管理**：按模块/场景划分错误码区间，便于排查与扩展。
- **可读可展示**：`message` 既能直接展示给用户（适当优化文案），也能辅助开发定位问题。

---

## 2. 错误码格式与区间划分

### 2.1 格式约定

- 类型：`int`
- 约定：
  - `0`：成功
  - `> 0`：业务错误
  - `< 0`：极少数系统级异常保留（如全局兜底），一般不使用，优先使用业务错误码

### 2.2 区间划分（建议）

| 区间        | 模块/含义             |
| ----------- | --------------------- |
| 1000 ~ 1999 | 通用错误              |
| 2000 ~ 2999 | 鉴权/账号相关（Auth） |
| 3000 ~ 3999 | 商家/分类（Shop）     |
| 4000 ~ 4999 | 推荐/每日推荐         |
| 5000 ~ 5999 | 评分（Rating）        |
| 6000 ~ 6999 | 收藏（Favorite）      |
| 9000 ~ 9999 | 其他/保留             |

> 说明：若后续增加“二手交易”等模块，可单独预留新区间。

## 3. 通用错误码（1000 ~ 1999）

| 错误码 | 标识名                    | 场景说明                                 | 建议前端处理                             |
| ------ | ------------------------- | ---------------------------------------- | ---------------------------------------- |
| 0      | `OK`                      | 成功                                     | 正常流程                                 |
| 1000   | `ERR_PARAM_INVALID`       | 参数不合法（缺失、格式错误、校验失败等） | 弹出提示文案，前端可结合字段高亮         |
| 1001   | `ERR_RESOURCE_NOT_FOUND`  | 数据不存在（商家、分类、配置等）         | 友好提示“数据不存在或已删除”             |
| 1002   | `ERR_METHOD_NOT_ALLOWED`  | 不支持的请求方式                         | 一般仅日志记录，前端展示通用错误         |
| 1003   | `ERR_BAD_REQUEST`         | 业务语义错误，无法处理                   | 展示错误文案或重试                       |
| 1004   | `ERR_DUPLICATE_OPERATION` | 重复操作（如重复收藏、重复评分提交等）   | 可静默处理或轻提示“已处理，请勿重复操作” |
| 1005   | `ERR_CONFLICT_STATUS`     | 状态冲突（如对已下架商家执行非法操作）   | 提示刷新页面或状态已变更                 |
| 1010   | `ERR_SYSTEM_BUSY`         | 系统繁忙/未知异常（兜底）                | 展示“系统繁忙，请稍后再试”，上报日志     |

## 4. 鉴权/账号错误码（2000 ~ 2999）

| 错误码 | 标识名                   | 场景说明                               | 建议前端处理                         |
| ------ | ------------------------ | -------------------------------------- | ------------------------------------ |
| 2000   | `ERR_UNAUTHORIZED`       | 未登录或 token 缺失                    | 跳转登录页或触发小程序静默登录       |
| 2001   | `ERR_TOKEN_INVALID`      | token 非法/无法解析                    | 清理本地 token，重新登录             |
| 2002   | `ERR_TOKEN_EXPIRED`      | token 过期                             | 提示登录过期并重新登录               |
| 2003   | `ERR_FORBIDDEN`          | 没有权限访问（如非管理员访问后台接口） | 提示“无权限访问”，跳转到首页或前一页 |
| 2010   | `ERR_ADMIN_LOGIN_FAILED` | 管理员登录失败（用户名或密码错误）     | 提示账号或密码错误                   |
| 2011   | `ERR_ADMIN_DISABLED`     | 管理员账号被禁用                       | 提示账号已被禁用                     |

## 5. 商家/分类模块错误码（3000 ~ 3999）

| 错误码 | 标识名                    | 场景说明                                       | 建议前端处理                         |
| ------ | ------------------------- | ---------------------------------------------- | ------------------------------------ |
| 3000   | `ERR_SHOP_NOT_FOUND`      | 商家不存在或已删除                             | 提示“商家不存在或已下架”，跳回列表页 |
| 3001   | `ERR_SHOP_OFFLINE`        | 商家已下架，不允许下单或部分操作               | 隐藏/禁用“去下单”按钮，提示已下架    |
| 3002   | `ERR_SHOP_INVALID_STATUS` | 商家状态不符合当前操作（如重复上架、重复下架） | 轻提示“状态已更新，请刷新后重试”     |
| 3003   | `ERR_SHOP_LINK_REQUIRED`  | 上架前未配置任何有效平台链接                   | 后台表单提示“请至少配置一个有效链接” |
| 3010   | `ERR_CATEGORY_NOT_FOUND`  | 分类不存在或已删除                             | 提示“分类不存在或已删除”             |
| 3011   | `ERR_CATEGORY_IN_USE`     | 分类被商家引用，禁止删除                       | 后台弹窗提示“分类被使用，无法删除”   |

## 6. 每日推荐模块错误码（4000 ~ 4999）

| 错误码 | 标识名                              | 场景说明                                       | 建议前端处理                         |
| ------ | ----------------------------------- | ---------------------------------------------- | ------------------------------------ |
| 4000   | `ERR_RECOMMEND_CONFIG_NOT_FOUND`    | 当日无推荐配置（后台未配置且随机数量配置缺失） | 小程序端可隐藏推荐区块或展示默认文案 |
| 4001   | `ERR_RECOMMEND_MODE_INVALID`        | 推荐模式非法（既不是手动也不是随机）           | 日志报警，前端展示通用错误           |
| 4002   | `ERR_RECOMMEND_MANUAL_SHOP_INVALID` | 手动推荐配置中包含不存在/下架商家              | 后台保存时给出具体错误提示           |

## 7. 评分模块错误码（5000 ~ 5999）

| 错误码 | 标识名                         | 场景说明                                      | 建议前端处理               |
| ------ | ------------------------------ | --------------------------------------------- | -------------------------- |
| 5000   | `ERR_RATING_SCORE_INVALID`     | 评分分值非法（非 1～5 整数）                  | 弹出表单校验错误，禁止提交 |
| 5001   | `ERR_RATING_SHOP_NOT_RATEABLE` | 当前商家不允许评分（如已删除/被禁用评分功能） | 提示“当前商家暂不支持评分” |

## 8. 收藏模块错误码（6000 ~ 6999）

| 错误码 | 标识名                        | 场景说明                                               | 建议前端处理                     |
| ------ | ----------------------------- | ------------------------------------------------------ | -------------------------------- |
| 6000   | `ERR_FAVORITE_SHOP_NOT_FOUND` | 收藏/取消收藏的商家不存在或已删除                      | 前端同步移除该收藏项并提示       |
| 6001   | `ERR_FAVORITE_ALREADY_EXISTS` | 已收藏再次收藏（后端也可视作幂等并返回成功，仅记日志） | 前端可忽略，保持收藏状态为已收藏 |
| 6002   | `ERR_FAVORITE_NOT_EXISTS`     | 取消收藏时记录不存在（后端也可视作幂等）               | 前端可忽略，保持收藏状态为未收藏 |

## 9. 使用约定与开发规范

- **统一封装**：后端定义统一响应模型，如：
  - `code`：int，错误码
  - `message`：String，错误信息
  - `data`：业务数据
- **统一异常**：
  - 业务异常使用自定义异常类（如 `BizException`），包含 `code` 与 `message`。
  - 全局异常处理器捕获后按本规范返回。
- **前端对照表**：
  - 小程序与后台前端可维护一个简单的 `errorCodeMap`，统一将关键错误码映射为用户可读文案。
  - 极少数需要特别处理的码（如 `ERR_UNAUTHORIZED`、`ERR_TOKEN_EXPIRED`）写死在拦截器逻辑中。

---

## 10. 扩展与维护

- 新增错误码时需：
  1. 在本文件中登记（编号、标识名、含义、建议前端处理）。
  2. 在后端统一常量类中添加常量（如 `ErrorCodes`）。
  3. 通知前端同学同步更新 errorCode 映射。
- 禁止在代码中直接写数字常量（如 `throw new BizException(4001, "...")`），必须通过统一常量引用。
