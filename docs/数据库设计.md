# 校园外卖信息聚合平台 — 数据库设计说明（外卖模块）

> 说明：本文件仅覆盖「外卖聚合模块」相关表结构，不含二手交易部分。数据库推荐使用 **MySQL 8.0 + InnoDB + utf8mb4**。

---

## 1. 设计原则

- **字符集**：`utf8mb4`，排序规则建议 `utf8mb4_general_ci`（或团队统一规范）。
- **存储引擎**：全部使用 `InnoDB`。
- **主键策略**：业务表统一使用 `BIGINT` 主键（可自增或雪花算法）。示例以 `BIGINT` 表示。
- **通用字段**：
  - `create_time`：DATETIME，创建时间。
  - `update_time`：DATETIME，更新时间。
  - `deleted`：TINYINT(1)，逻辑删除标记（0=正常，1=删除）。
- **命名约定**：
  - 表名：小写下划线，如 `shop`, `shop_rating`。
  - 索引名：以 `idx_`（普通索引）或 `uk_`（唯一索引）开头。

## 2. 用户与管理员

### 2.1 小程序用户表 `user`

用于标识微信小程序端的普通用户（学生），为评分、收藏等行为提供主体。

#### 字段设计

| 字段名            | 类型         | 允许空 | 说明                      |
| ----------------- | ------------ | ------ | ------------------------- |
| `id`              | BIGINT       | 否     | 主键 ID                   |
| `openid`          | VARCHAR(64)  | 否     | 微信 openid，用户唯一标识 |
| `unionid`         | VARCHAR(64)  | 是     | 微信 unionid，可空        |
| `nickname`        | VARCHAR(64)  | 是     | 昵称                      |
| `avatar`          | VARCHAR(255) | 是     | 头像 URL                  |
| `status`          | TINYINT(1)   | 否     | 状态：1=正常，0=禁用      |
| `last_login_time` | DATETIME     | 是     | 最近登录时间              |
| `create_time`     | DATETIME     | 否     | 创建时间                  |
| `update_time`     | DATETIME     | 否     | 更新时间                  |
| `deleted`         | TINYINT(1)   | 否     | 逻辑删除：0=正常，1=删除  |

#### 索引设计

- 唯一索引：`uk_user_openid (openid)`
- 普通索引：`idx_user_create_time (create_time)`

用于后台管理端登录的管理员账号。

#### 字段设计

| 字段名            | 类型         | 允许空 | 说明                 |

| `last_login_time` | DATETIME     | 是     | 最近登录时间         |
| `create_time`     | DATETIME     | 否     | 创建时间             |
| `update_time`     | DATETIME     | 否     | 更新时间             |
| `deleted`         | TINYINT(1)   | 否     | 逻辑删除             |

- 唯一索引：`uk_admin_username (username)`

## 3. 商家与分类

### 3.1 商家分类表 `shop_category`

用于维护一级分类，如：快餐、奶茶、正餐、夜宵、其他等。

#### 字段设计

| 字段名        | 类型         | 允许空 | 说明                 |
| ------------- | ------------ | ------ | -------------------- |
| `id`          | BIGINT       | 否     | 主键 ID              |
| `name`        | VARCHAR(32)  | 否     | 分类名称             |
| `logo_url`    | VARCHAR(255) | 是     | 分类 Logo/图标 URL   |
| `sort_order`  | INT          | 否     | 排序值，越大越靠前   |
| `status`      | TINYINT(1)   | 否     | 状态：1=启用，0=禁用 |
| `create_time` | DATETIME     | 否     | 创建时间             |
| `update_time` | DATETIME     | 否     | 更新时间             |
| `deleted`     | TINYINT(1)   | 否     | 逻辑删除             |

#### 索引设计

- 普通索引：`idx_category_sort (sort_order)`
- 可选唯一索引：`uk_category_name (name)`

---

### 3.2 商家表 `shop`

对应需求文档中的商家核心信息及评分统计。

#### 字段设计

| 字段名              | 类型          | 允许空 | 说明                              |
| ------------------- | ------------- | ------ | --------------------------------- |
| `id`                | BIGINT        | 否     | 主键 ID                           |
| `name`              | VARCHAR(60)   | 否     | 商家名称（2～30 字）              |
| `category_id`       | BIGINT        | 否     | 分类 ID，关联 `shop_category.id`  |
| `cover_url`         | VARCHAR(255)  | 是     | 封面图 / Logo URL                 |
| `introduction`      | VARCHAR(200)  | 是     | 商家介绍（用于首页小字展示）      |
| `business_hours`    | VARCHAR(128)  | 是     | 营业时间说明                      |
| `delivery_scope`    | VARCHAR(128)  | 是     | 配送范围说明                      |
| `starting_price`    | DECIMAL(10,2) | 是     | 起送价                            |
| `delivery_fee_desc` | VARCHAR(64)   | 是     | 配送费说明                        |
| `sort_weight`       | INT           | 否     | 排序权重，默认 0                  |
| `status`            | TINYINT(1)    | 否     | 状态：1=上架，0=下架              |
| `campus_code`       | VARCHAR(32)   | 否     | 校区标识，单校区可固定一个值      |
| `avg_score`         | DECIMAL(3,2)  | 否     | 平均评分（1.00～5.00），默认 0.00 |
| `rating_count`      | INT           | 否     | 评分人数，默认 0                  |
| `remark`            | VARCHAR(255)  | 是     | 后台备注，前台不展示              |
| `create_time`       | DATETIME      | 否     | 创建时间                          |
| `update_time`       | DATETIME      | 否     | 更新时间                          |
| `deleted`           | TINYINT(1)    | 否     | 逻辑删除                          |

#### 索引设计

- 普通索引：`idx_shop_category_status (category_id, status)`
- 普通索引：`idx_shop_status_update (status, update_time)`
- 普通索引：`idx_shop_sort_weight (sort_weight DESC)`
- 普通索引：`idx_shop_name (name)`（支持名称模糊查询）

---

### 3.3 商家平台链接表 `shop_platform_link`

一商家多链接（美团、饿了么、京东、小程序自营等）的配置。

#### 字段设计

| 字段名               | 类型         | 允许空 | 说明                                                     |
| -------------------- | ------------ | ------ | -------------------------------------------------------- |
| `id`                 | BIGINT       | 否     | 主键 ID                                                  |
| `shop_id`            | BIGINT       | 否     | 商家 ID，关联 `shop.id`                                  |
| `platform_type`      | TINYINT      | 否     | 所属平台：1=美团，2=饿了么，3=京东，4=小程序自营，9=其他 |
| `link_type`          | TINYINT      | 否     | 链接类型：1=H5，2=外部小程序，3=内部小程序 path          |
| `h5_url`             | VARCHAR(512) | 是     | H5 跳转链接，`link_type=1` 时使用                        |
| `mini_program_appid` | VARCHAR(64)  | 是     | 外部小程序 appid，`link_type=2` 时使用                   |
| `mini_program_path`  | VARCHAR(255) | 是     | 小程序 path，自营或外部均可使用                          |
| `is_primary`         | TINYINT(1)   | 否     | 是否主推入口：1=是，0=否，默认 0                         |
| `status`             | TINYINT(1)   | 否     | 状态：1=有效，0=失效                                     |
| `create_time`        | DATETIME     | 否     | 创建时间                                                 |
| `update_time`        | DATETIME     | 否     | 更新时间                                                 |
| `deleted`            | TINYINT(1)   | 否     | 逻辑删除                                                 |

#### 索引设计

- 普通索引：`idx_platform_shop (platform_type, shop_id)`
- 普通索引：`idx_link_shop (shop_id)`

## 4. 评分与收藏

### 4.1 商家评分表 `shop_rating`

记录用户对商家的星级评分（1～5 整星），一用户一商家仅保留一条评分记录。

#### 字段设计

| 字段名        | 类型       | 允许空 | 说明                     |
| ------------- | ---------- | ------ | ------------------------ |
| `id`          | BIGINT     | 否     | 主键 ID                  |
| `user_id`     | BIGINT     | 否     | 用户 ID，关联 `user.id`  |
| `shop_id`     | BIGINT     | 否     | 商家 ID，关联 `shop.id`  |
| `score`       | TINYINT    | 否     | 星级评分：1～5           |
| `create_time` | DATETIME   | 否     | 创建时间（首次评分时间） |
| `update_time` | DATETIME   | 否     | 更新时间（最近评分时间） |
| `deleted`     | TINYINT(1) | 否     | 逻辑删除                 |

#### 索引设计

- 唯一索引：`uk_rating_user_shop (user_id, shop_id)` —— 一用户一商家一条评分
- 普通索引：`idx_rating_shop (shop_id)`
- 普通索引：`idx_rating_user (user_id)`

> 说明：更新评分时，同步更新 `shop.avg_score` 与 `shop.rating_count`。

---

### 4.2 商家收藏表 `shop_favorite`

支持用户收藏商家与「我的收藏」快速跳转。

#### 字段设计

| 字段名        | 类型       | 允许空 | 说明                             |
| ------------- | ---------- | ------ | -------------------------------- |
| `id`          | BIGINT     | 否     | 主键 ID                          |
| `user_id`     | BIGINT     | 否     | 用户 ID，关联 `user.id`          |
| `shop_id`     | BIGINT     | 否     | 商家 ID，关联 `shop.id`          |
| `create_time` | DATETIME   | 否     | 收藏时间                         |
| `deleted`     | TINYINT(1) | 否     | 逻辑删除（也可选择物理删除实现） |

#### 索引设计

- 唯一索引：`uk_favorite_user_shop (user_id, shop_id)` —— 一用户一商家一条收藏
- 普通索引：`idx_favorite_user (user_id, create_time DESC)` —— 我的收藏列表
- 普通索引：`idx_favorite_shop (shop_id)`

## 5. 每日推荐配置

### 5.1 配置表 `daily_recommend_config`

按自然日存储每日推荐规则（手动/随机）与手动推荐时的商家列表。

#### 字段设计

| 字段名            | 类型     | 允许空 | 说明                                                 |
| ----------------- | -------- | ------ | ---------------------------------------------------- |
| `id`              | BIGINT   | 否     | 主键 ID                                              |
| `biz_date`        | DATE     | 否     | 业务日期（自然日，如 2025-02-09），同一天仅一条记录  |
| `mode`            | TINYINT  | 否     | 模式：1=手动，2=随机                                 |
| `manual_shop_ids` | TEXT     | 是     | 手动模式时的商家 ID 列表（JSON 数组），mode=1 时使用 |
| `random_count`    | INT      | 是     | 随机模式展示数量 N，默认 4                           |
| `create_time`     | DATETIME | 否     | 创建时间                                             |
| `update_time`     | DATETIME | 否     | 更新时间                                             |

#### 索引设计

- 唯一索引：`uk_daily_biz_date (biz_date)` —— 一天一条配置

> 使用建议：小程序端请求每日推荐时，根据当前日期查询该表，按 mode 选择手动列表或随机生成，并可结合 Redis 做结果缓存。

## 6. 操作日志

### 6.1 操作日志表 `operation_log`

记录管理员在后台的关键操作，满足审计与排查需求。

#### 字段设计

| 字段名         | 类型         | 允许空 | 说明                                             |
| -------------- | ------------ | ------ | ------------------------------------------------ |
| `id`           | BIGINT       | 否     | 主键 ID                                          |
| `admin_id`     | BIGINT       | 否     | 管理员 ID，关联 `admin_user.id`                  |
| `action`       | VARCHAR(64)  | 否     | 操作类型编码，如 `SHOP_CREATE`, `SHOP_UPDATE` 等 |
| `target_type`  | VARCHAR(32)  | 否     | 目标类型，如 `SHOP`, `CATEGORY`                  |
| `target_id`    | BIGINT       | 是     | 目标对象 ID                                      |
| `request_data` | TEXT         | 是     | 操作请求数据（JSON），建议只存必要业务字段       |
| `diff_data`    | TEXT         | 是     | 变更前后差异（可选）                             |
| `ip`           | VARCHAR(64)  | 是     | 操作 IP                                          |
| `user_agent`   | VARCHAR(255) | 是     | UA 信息，可选                                    |
| `create_time`  | DATETIME     | 否     | 创建时间                                         |

#### 索引设计

- 索引：`idx_log_admin_time (admin_id, create_time DESC)`
- 索引：`idx_log_target (target_type, target_id)`

## 7. 表关系总览（文字版）

- `user` 与：
  - `shop_rating`：一对多（一个用户可对多个商家评分）；
  - `shop_favorite`：一对多（一个用户可收藏多个商家）。
- `admin_user` 与：
  - `operation_log`：一对多。
- `shop_category` 与 `shop`：一对多。
- `shop` 与：
  - `shop_platform_link`：一对多；
  - `shop_rating`：一对多；
  - `shop_favorite`：一对多。
- `daily_recommend_config`：按自然日一行，与具体商家通过 ID 列表（或随机规则）关联。

## 8. 后续扩展建议

- 若后续需要**多校区**，可在 `shop`、`user` 等表增加 `school_id` / `campus_id` 字段，并为其建立索引。
- 若需要更复杂的运营统计（如按分类、评分、收藏排行等），建议增加独立的统计表或离线数仓，而不是在业务表上做重型统计。
- 若操作日志量较大，可单独建库或按时间分表。
