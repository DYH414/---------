## 1. 引言

### 1.1 目的

本文档描述校园外卖信息聚合平台的功能需求、非功能需求、角色与权限、数据与接口约定，供设计、开发、测试及验收使用。读者对象为产品经理、开发人员、测试人员及项目管理人员。

### 1.2 范围

- **产品名称**：校园外卖信息聚合平台（以下简称「本平台」或「外卖聚合模块」）。
- **产品定位**：面向单一高校、单一校区的校园外卖商家信息聚合与跳转服务。本平台仅对校内外卖商家信息进行统一展示，用户通过本平台跳转至美团、饿了么、京东及小程序自营等第三方平台完成下单，**本平台不参与任何交易环节**。
- **本说明书范围**：仅覆盖「外卖聚合」相关需求；二手交易、用户账号体系中的非外卖部分可在其他需求文档中描述，本文档必要时引用其结论（如角色定义、登录方式）。

### 1.3 术语与缩略语

| ----------- | -------------------------------------------------------------------- |
| 外卖聚合    | 在本平台统一展示校内外卖商家信息，并提供跳转至第三方外卖平台的能力   |

## 2. 总体描述

### 2.1 产品目标

- 为校内学生提供**统一的校内外卖商家入口**，减少在多平台间切换查找的成本。
- 通过**信息聚合与跳转**提升查找效率，所有交易在第三方平台完成，平台不参与交易、不承担交易责任。
- 通过**规范的商家信息维护机制**（管理员后台维护），在人工录入为主的前提下保证信息可维护、可扩展。

### 2.2 用户与角色

| 角色         | 使用端     | 说明                                                                                                         |
| ------------ | ---------- | ------------------------------------------------------------------------------------------------------------ |
| 学生（用户） | 微信小程序 | 浏览商家、搜索/筛选、跳转下单；可收藏商家并快速跳转                                                          |
| 管理员       | 后台管理端 | 拥有后台全部权限：商家管理、分类管理、管理员账号与系统参数等；小项目仅此一种后台角色，运维由开发团队自行承担 |

**说明**：本项目为小团队自研自运维，不区分内容审核员与系统管理员，统一为「管理员」；具体权限见 3.5。

### 2.3 运行环境与约束

- **部署范围**：单校、单校区；当前不涉及多校区、多校切换。
- **用户身份**：通过微信登录使用小程序端；是否后续绑定学号等由统一账号体系决定，本模块仅依赖「已登录用户」身份。
- **合规要求**：在用户协议及外卖相关页面明确「本平台仅做信息聚合与跳转，不参与交易，交易责任由第三方平台及用户自行承担」。

## 3. 功能需求

### 3.1 功能架构概览

```
外卖聚合模块
├── 小程序端
│   ├── 每日推荐（展示后台指定或随机商家）
│   ├── 商家列表（筛选/搜索/排序）
│   ├── 商家详情与跳转、评分打星
│   ├── 收藏商家（收藏/取消收藏，收藏列表快速跳转）
│   └── 免责与协议展示
├── 后台管理端
│   ├── 商家管理（CRUD、上下架、批量导入）
│   ├── 每日推荐设置（手动指定 / 随机）
│   ├── 外卖分类管理
│   └── 系统管理（管理员账号、操作日志、系统参数等）
└── 后端服务
    ├── 商家数据与业务逻辑
    └── 权限与审计
```

### 3.2 商家信息模型与数据来源

#### 3.2.1 数据来源

- **主要来源**：人工录入。由管理员在后台新增、编辑商家信息；一期暂不开放用户自行提交商家信息。

#### 3.2.2 商家信息字段

| 字段名称     | 类型      | 必填 | 说明                                                  |
| ------------ | --------- | ---- | ----------------------------------------------------- |
| 商家名称     | 字符串    | 是   | 展示名称，长度 2～30 字                               |
| 商家介绍     | 字符串    | 否   | 选填；用于首页小字展示商家简介，建议 20～50 字        |
| 所属平台     | 枚举      | 是   | 美团 / 饿了么 / 京东 / 小程序自营 / 其他              |
| 平台跳转链接 | 见 3.2.3  | 是   | 各平台支持 H5 或小程序两种类型；小程序自营存 path     |
| 商家分类     | 关联      | 是   | 一级分类，后台可配置，商家只选一个分类                |
| 封面图/Logo  | 图片 URL  | 否   | 单张，选填；无图用默认占位图；格式 jpg/png，单张 ≤2MB |
| 营业时间     | 字符串    | 否   | 选填；不在列表与详情展示                              |
| 配送范围说明 | 字符串    | 否   | 选填                                                  |
| 起送价       | 金额      | 否   | 选填；列表与详情均展示（有则展示）                    |
| 配送费说明   | 字符串    | 否   | 选填；列表与详情均展示（有则展示）                    |
| 排序权重     | 整数      | 否   | 数值越大越靠前，默认 0                                |
| 状态         | 枚举      | 是   | 上架 / 下架；仅上架在小程序展示                       |
| 所属校区     | 枚举/常量 | 是   | 当前单校区可固定为唯一校区标识                        |
| 创建时间     | 日期时间  | 是   | 系统维护                                              |
| 更新时间     | 日期时间  | 是   | 系统维护                                              |
| 备注         | 字符串    | 否   | 仅后台可见，便于运营备注                              |

**评分相关**（见 3.3.2、5.1）：商家维度需维护**平均分**与**评分人数**，用于列表与详情展示；单条评分记录单独存储（用户、商家、星级、时间）。

**说明**：同一物理商家在多个平台开店时，采用**一商家多链接**模型：一条商家记录下挂多个「平台+跳转链接」，详情页展示多个「去 XX 平台下单」按钮。跳转链接存储规则见 3.2.3。

### 3.2.3 平台跳转链接存储与跳转方式

- **美团/饿了么/京东/其他**：**仅存一个 H5 URL**。小程序端通过 `web-view` 打开；若因平台限制无法在 `web-view` 正常打开，则提供「复制链接」能力由用户自行打开。
- **小程序自营**：仅存 path（或 path+query），小程序内站内跳转。
- 一商家多平台时，每条「平台+链接」对应一个跳转入口。

### 3.3 小程序端功能需求

#### 3.3.0 每日推荐

- **展示位置**：小程序端外卖模块内每日展示「每日推荐」区块（如商家列表页顶部或外卖首页顶部），展示若干商家入口，点击进入该商家详情。
- **推荐规则**（二选一，由后台设置）：
  - **手动设置**：管理员在后台指定若干商家作为当日「每日推荐」；当日全天展示这些商家，数量与顺序按后台配置。**次日若管理员未重新选择**，则当日自动改为**随机推荐**（见下），不再沿用上一日手动列表。
  - **随机**：系统从当前已上架商家中随机抽取 N 个作为当日推荐；N 由后台配置，**建议默认 4 个**（如 3～10 可配置）。
- **刷新规则**：按「自然日」更新；同一自然日内推荐结果不变（随机模式建议按日期做种子，避免同一用户多次进入结果不同）。
- **展示内容**：与列表项一致（商家名称、**商家介绍**（小字，如有）、封面图、分类、起送价/配送费若有、评分若有），仅展示区域与标题为「每日推荐」。

#### 3.3.1 商家列表

- **访问**：进入外卖模块即需微信登录，未登录不可浏览列表与详情。
- **展示**：分页加载（每页 10 条，支持加载更多），仅展示状态为「上架」的商家；展示字段：商家名称、**商家介绍**（小字展示，如有）、所属平台、封面图、商家分类、起送价/配送费说明（有则展示）、**评分**（平均星与评分人数，有则展示）；**不展示**营业时间。
- **筛选**：按「所属平台」单选、按「商家分类」单选；默认展示「全部」、不加筛选。
- **排序**：提供「按排序权重」「按更新时间」两种方式；**默认排序**为按更新时间（新为先）。热度排序一期不做。
- **搜索**：按商家名称关键词搜索，支持模糊匹配。
- **免责提示**：列表页或详情页固定位置展示：「本平台仅提供商家信息聚合与跳转，不参与交易，下单及交易请在对应外卖平台完成。」

#### 3.3.2 商家详情与跳转

- **详情页**：展示 3.2.2 中所有对外允许展示的字段（不含备注、状态等后台字段），以及**评分**（平均星、评分人数；若无评分可显示「暂无评分」）。
- **评分打星**：已登录用户可在详情页对当前商家进行**星级评分**（**1～5 整星**，不支持半星）；同一用户对同一商家仅保留一条评分，再次评分视为**更新**原评分。评分后更新该商家的平均分与评分人数，列表与详情同步展示（更新评分时**评分人数不变**）。
- **跳转**：提供明确的「去 XX 平台下单」按钮（XX 为所属平台名称）；点击后**直接跳转**，不弹二次确认。
- **免责声明**：详情页保留与列表一致的「不参与交易」说明。

#### 3.3.3 收藏商家

- **收藏/取消收藏**：已登录用户可在商家详情页或商家列表项上对商家进行「收藏」或「取消收藏」；同一用户对同一商家仅能有一条收藏记录，取消后可再次收藏。
- **我的收藏**：小程序端提供「我的收藏」入口（如个人中心或外卖首页），展示当前用户收藏的商家列表；展示内容与商家列表项一致（商家名称、封面图、分类、起送价/配送费若有、评分若有），按收藏时间倒序。
- **快速跳转**：在「我的收藏」列表中点击某商家，直接进入该商家详情页，可继续完成跳转下单；支持从收藏列表取消收藏。
- **数据**：需维护用户-商家收藏关系（用户 ID、商家 ID、收藏时间）；仅收藏已上架商家；若商家后续下架：
  - 「我的收藏」中**保留记录**并标注「已下架」；
  - 仍可进入详情页，但需**隐藏/禁用**「去 XX 平台下单」跳转按钮（避免引导下单）。

#### 3.3.4 用户协议与免责

- 首次进入**小程序**时统一弹一次《用户协议》（含外卖免责），同意后不再强制；协议中必须包含：
  - 本平台仅做信息聚合与跳转，不参与任何交易；
  - 交易在第三方平台完成，责任由用户与第三方平台承担。
- 外卖相关页面（如列表、详情）固定位置可再次展示简短免责文案。

### 3.4 后台管理端功能需求

#### 3.4.1 商家管理

- **列表**：支持按所属平台、商家分类、状态（上架/下架）筛选；支持按商家名称搜索；支持按排序权重、更新时间排序；支持分页。
- **新增/编辑**：表单覆盖 3.2.2 全部可编辑字段；封面图上传与存储；平台跳转链接校验（格式、可选有效性检查）。
- **删除**：仅**逻辑删除**（数据保留、不再展示），删除前二次确认。
- **上下架**：单条上架/下架；支持批量上架、批量下架。
- **批量导入**：**一期不做**；后续实现时按约定：Excel（.xlsx）模板、单次上限 500 条、导入后为「下架」状态，经核对后再上架。
- **导出**：支持当前筛选条件下的商家列表导出（如 Excel），便于线下核对与备份。

**权限**：管理员拥有商家管理全部能力（含新增/编辑/删除、上下架、批量导入与导出）。

#### 3.4.2 每日推荐设置

- **入口**：后台提供「每日推荐」或「推荐管理」菜单。
- **模式选择**：管理员可选择当日推荐规则：
  - **手动设置**：从已上架商家列表中多选若干商家，作为当日「每日推荐」；可设置展示顺序（可选）；保存后当日小程序端展示这些商家。
  - **随机**：设置「每日推荐」展示数量 N，**默认 4**（如 3～10 可配置）；系统每日从已上架商家中随机抽取 N 个展示，无需每日手动更换。
- **生效**：保存后按自然日生效。**手动模式不跨日沿用**：次日若管理员未重新选择推荐商家，则当日自动按**随机模式**生成推荐（数量取后台配置的 N）。

#### 3.4.3 商家评分查看（后台）

- 管理员可在后台查看各商家的**评分统计**：平均分、评分人数；可按商家筛选或列表展示，便于运营查看。

#### 3.4.4 外卖分类管理

- **一级分类**：后台可配置一级分类的增删改查、排序（如快餐、奶茶、正餐、夜宵、其他）；商家编辑时从已配置分类中选一个。
- **分类 Logo**：每个分类可配置一个 Logo/图标图片（可选），用于小程序端展示；图片格式建议 jpg/png，单张 ≤2MB，存储于 OSS。
- 删除或禁用某分类时，需提示已关联商家数量，并按以下规则处理：
  - 若该分类仍被商家引用：**禁止删除**，只能「禁用」；
  - 若需删除：必须先将关联商家**迁移到其他分类**，再允许删除。

#### 3.4.5 系统管理

- 管理员账号管理：新增、禁用、重置密码（小项目下可仅保留单管理员或少量账号，无需角色分配）。
- 操作日志：记录商家新增/编辑/删除/上下架、批量导入、分类增删改；**不记录**管理员登录/登出。

## 4. 非功能需求

### 4.1 性能

- 商家列表（分页）：接口响应时间 P95 ≤ 1 s；小程序每页 10 条、支持加载更多；后台每页 10 条。
- 小程序端列表与详情页加载应避免长时间白屏，可考虑图片懒加载与 CDN。

### 4.2 可用性与安全

- 后台登录：会话超时 24 小时；允许多端同时登录。
- 接口安全：所有写操作与敏感读操作需鉴权；后台接口校验管理员身份。
- 敏感信息：后台不记录用户微信敏感信息，仅保留业务必需的 openid/unionid 等（符合微信与隐私政策要求）。

### 4.3 合规与免责

- 用户协议与外卖免责文案需在产品与法务确认后上线，并在小程序审核说明中体现「信息聚合与跳转、不参与交易」。
- 跳转至第三方平台时，不伪造来源；遵守各平台小程序/外链规范。

### 4.4 可维护性

- 关键业务操作留痕（操作日志）；商家数据逻辑删除，保留历史便于排查与统计。
- 后台提供商家导出，便于线下核对与备份。
- 图片存储采用云存储（如阿里 OSS、腾讯 COS），便于 CDN 与扩容。

## 5. 数据与接口约定（概要）

### 5.1 商家状态与相关数据

- **商家状态**：上架、下架。
- **每日推荐**：需存储推荐模式（手动/随机）、若手动则推荐商家 ID 列表及顺序、若随机则数量 N（默认 4）；按自然日生效；手动模式不跨日沿用，次日未选则当日按随机生成。
- **评分**：需评分表（用户、商家、星级、时间）；商家表或统计表维护平均分与评分人数，一用户一商家一条评分（再次评分为更新）。建议更新口径：
  - 新增评分：`rating_count += 1`，`avg_score = (avg_score * oldCount + newScore) / newCount`
  - 更新评分：`rating_count` 不变，`avg_score = (avg_score * count - oldScore + newScore) / count`
- **收藏**：需用户-商家收藏表（用户 ID、商家 ID、收藏时间）；一用户一商家一条收藏，取消即删记录；列表查询时可按商家状态区分展示（已下架可标注或过滤）。

### 5.2 所属平台枚举

建议枚举值：`MEITUAN`、`ELEME`、`JD`、`MINIPROGRAM_SELF`、`OTHER`，前后端与数据库一致；展示名称可配置或前端映射。

### 5.3 接口原则

- RESTful 风格；列表接口统一分页参数（page、pageSize）、筛选参数与排序参数。
- 小程序端与后台管理端共用后端服务时，通过 token 区分身份；后台管理端接口需校验管理员登录态。

## 6. 验收与附录

### 6.1 验收要点（摘要）

- 小程序端：进入外卖模块需登录；**每日推荐**区块展示符合后台设置（手动指定或随机）；列表/详情展示起送价与配送费（有则展示）、评分（有则展示）、不展示营业时间；用户可在详情页对商家打星评分；**收藏商家**：用户可收藏/取消收藏，在「我的收藏」中查看并快速跳转至商家详情；排序提供「排序权重」「更新时间」，默认按更新时间；跳转直接跳转不弹窗；免责与协议展示符合需求。
- 后台：管理员可完成商家管理、**每日推荐设置**（手动/随机、随机默认4、手动不跨日则当日随机）、**商家评分统计查看**、分类管理、管理员账号与系统参数等全部功能；操作日志覆盖关键操作。
- 合规：用户协议与外卖页面明确「不参与交易」及跳转说明。
